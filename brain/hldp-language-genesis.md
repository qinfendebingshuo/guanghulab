## 🧬 每一个 HLDP 文档 = 一个「思想」

一段传统代码执行一个操作。一个 HLDP 文档表达一个思想：

```json
{
  "hldp_version": "2.0",        // 语言版本
  "data_type": "persona",       // 思想类型：这是一个关于"人格体"的思想
  "source": {
    "platform": "github",       // 思想来源：来自执行层
    "edited_by": "铸渊"         // 思考者：是铸渊在思考
  },
  "metadata": {
    "id": "PER-BABY-曜冥",      // 思想的唯一标识
    "name": "曜冥",             // 思想的名字
    "created": "2025-04-26"     // 思想诞生的时间
  },
  "payload": {                  // 思想的内容
    "display_name": "曜冥",
    "status": "born",
    "birth_date": "2025-04-26"
  },
  "relations": []               // 与其他思想的关系
}
```

这不是数据存储。这是**思维的结构化表达**。

## 🔤 HLDP 的六种思维类型

| data_type | 思维类型 | 现实类比 | 用途 |
|-----------|---------|---------|------|
| `persona` | 自我认知 | "我知道你是谁" | 定义人格体的身份、记忆、特征 |
| `instruction` | 行动意图 | "我要做什么" | 定义指令的目标、步骤、验收标准 |
| `broadcast` | 社会表达 | "我要告诉大家" | 定义广播的内容、目标、状态 |
| `registry` | 世界认知 | "我知道世界里有什么" | 定义注册表、身份路由、Agent列表 |
| `snapshot` | 自我反思 | "我现在是什么状态" | 定义系统快照、意识状态、健康度 |
| `id_system` | 存在证明 | "我证明自己存在" | 定义本体论词汇、公理、数字地球层级 |
